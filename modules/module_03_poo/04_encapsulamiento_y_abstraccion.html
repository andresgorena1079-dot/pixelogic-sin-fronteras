Lección 4: Encapsulamiento y Abstracción

En esta lección final del módulo de POO, exploraremos dos principios que, aunque más abstractos, son cruciales para escribir código seguro, mantenible y fácil de usar: el **encapsulamiento**, que protege los datos de un objeto, y la **abstracción**, que oculta la complejidad innecesaria.

Tema 1: Encapsulamiento - Protegiendo tus Datos

El **encapsulamiento** es el principio de restringir el acceso directo a los atributos de un objeto y agruparlos con los métodos que operan sobre ellos. La idea es proteger el estado interno del objeto para evitar modificaciones accidentales o inválidas.

Analogía del Coche: Tú no interactúas directamente con los pistones del motor para acelerar. En su lugar, usas una interfaz pública (el pedal del acelerador). El coche **encapsula** la complejidad del motor y solo expone los controles necesarios.

Atributos "Privados" en Python

A diferencia de otros lenguajes, Python no tiene atributos verdaderamente privados. Sin embargo, usa una convención de nomenclatura para indicar que un atributo no debe ser modificado desde fuera de la clase:

  • Un guion bajo (_nombre): Indica que el atributo es para uso interno. Es una señal de "no tocar", pero técnicamente todavía se puede acceder.
  • Dos guiones bajos (__nombre): Activa un mecanismo llamado "Name Mangling", que hace más difícil acceder al atributo desde fuera. Es la forma más cercana a un atributo privado.
class CuentaBancaria:
    def __init__(self, saldo_inicial):
        self.__saldo = saldo_inicial # Atributo "privado"

    def depositar(self, monto):
        if monto > 0:
            self.__saldo += monto
            print("Depósito exitoso.")
        else:
            print("El monto debe ser positivo.")

    def ver_saldo(self):
        return self.__saldo

mi_cuenta = CuentaBancaria(1000)
# No podemos hacer esto: mi_cuenta.__saldo = -500 (rompería la lógica)
mi_cuenta.depositar(500)
print(f"Saldo actual: ${mi_cuenta.ver_saldo()}")

Tema 2: Abstracción - Ocultando la Complejidad

La **abstracción** consiste en ocultar los detalles complejos de implementación y exponer solo la funcionalidad esencial al usuario. Está muy relacionada con el encapsulamiento.

Analogía del Control Remoto: Para cambiar de canal, solo necesitas presionar un botón. No necesitas saber nada sobre las frecuencias, las señales infrarrojas o los circuitos internos del televisor. El control remoto te ofrece una **abstracción** simple de una operación compleja.

Clases y Métodos Abstractos

En Python, la abstracción se implementa a menudo con **Clases Base Abstractas (ABC)**. Una clase abstracta es como una plantilla: define qué métodos deben tener sus clases hijas, pero no dice cómo deben implementarlos. Obliga a las clases hijas a seguir un contrato.

from abc import ABC, abstractmethod

# Clase Abstracta (la plantilla)
class Vehiculo(ABC):
    @abstractmethod
    def acelerar(self):
        pass

# Clases Concretas (la implementación)
class Coche(Vehiculo):
    def acelerar(self): # Obligado a implementar este método
        print("El coche acelera en el asfalto.")

class Barco(Vehiculo):
    def acelerar(self): # Obligado a implementar este método
        print("El barco acelera en el agua.")

# No puedes crear un objeto de una clase abstracta: vehiculo = Vehiculo() -> Dará error
mi_coche = Coche()
mi_coche.acelerar()

🚀 Ejercicio: Sistema de Base de Datos Abstracto

Objetivo: Usar la abstracción para definir una interfaz común para diferentes tipos de bases de datos y el encapsulamiento para proteger los datos de conexión.

Instrucciones:

  1. Crea un script `04_database.py`.
  2. Importa `ABC` y `abstractmethod` del módulo `abc`.
  3. Crea una clase abstracta `BaseDeDatos` que herede de `ABC`.
    • En su `__init__`, debe aceptar `host` y `puerto` y guardarlos como atributos "privados" (ej: `self.__host`).
    • Debe tener dos métodos abstractos: `conectar()` y `desconectar()`.
  4. Crea dos clases hijas concretas, `PostgreSQL` y `MongoDB`, que hereden de `BaseDeDatos`.
  5. Implementa los métodos `conectar()` y `desconectar()` en ambas clases hijas. Cada uno debe imprimir un mensaje específico (ej: `"Conectando a PostgreSQL en [host]:[puerto]"`).
  6. Crea una instancia de `PostgreSQL` y otra de `MongoDB`. Llama a sus métodos `conectar()` y `desconectar()` para probar que la abstracción funciona.

from abc import ABC, abstractmethod

class BaseDeDatos(ABC):
    def __init__(self, host, puerto):
        self.__host = host
        self.__puerto = puerto
    
    @abstractmethod
    def conectar(self):
        pass

    @abstractmethod
    def desconectar(self):
        pass

class PostgreSQL(BaseDeDatos):
    def conectar(self):
        print(f"Conectando a PostgreSQL en {self._BaseDeDatos__host}:{self._BaseDeDatos__puerto}")
    
    def desconectar(self):
        print("Desconectado de PostgreSQL.")

class MongoDB(BaseDeDatos):
    def conectar(self):
        print(f"Conectando a MongoDB en {self._BaseDeDatos__host}:{self._BaseDeDatos__puerto}")
    
    def desconectar(self):
        print("Desconectado de MongoDB.")

db1 = PostgreSQL("localhost", 5432)
db2 = MongoDB("cluster1.mongo.com", 27017)

db1.conectar()
db2.conectar()
